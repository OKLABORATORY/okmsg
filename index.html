Overview

OKMessaging is a powerful, AI-powered CPaaS platform built on a modern microservices architecture using Docker and Kubernetes, delivering unmatched scalability, performance, and intelligent omni-channel communication across SMS, voice, and WhatsApp.

The platform enables businesses to send bulk voice calls using pre-recorded audio or advanced text-to-speech (TTS) robotic voices. With intuitive campaign management, users can easily schedule, launch, and manage high-volume communication workflows.

OKMessaging supports a secure multi-tenant architecture, allowing organizations to create and manage client accounts while enabling service providers to monetize the platform via subscription or usage-based models.

From contact uploads to analytics and delivery tracking, OKMessaging provides a centralized, scalable, and enterprise-ready messaging solution.

üîç Platform Overview

The SMPP messaging platform runs entirely inside a Docker-based container environment. No service is allowed to operate outside the container.

All core services are started in parallel. The platform initializes the SMPP Server and SMPP Client as the foundation.

The SMPP Server opens multiple listening ports configured via the .env file. Each port is dedicated to accepting authenticated SMPP client connections.

Clients authenticate using valid credentials (Username, Password, IP, Port). Any invalid authentication results in an immediate disconnection.

1
SMPP_CLIENT Service

Prepares and submits messages after successful authentication.

  • Validates message parameters
  • Identifies channel (S / H / A)
  • Handles encoding & message splitting
  • Attaches optional TLVs
2
SMPP_SERVER Service

Performs security and throughput checks before message acceptance.

  • Generates unique OMSGID
  • Revalidates credentials
  • Applies rate & capacity limits
3
DF Service (DragonFly / Redis)

Acts as the core message queue and persistence layer.

  • Secure message storage
  • Metadata preservation
  • Queue reliability
4
Validator Service

Ensures compliance, billing readiness, and content integrity.

  • Billing validation
  • Blacklist checks
  • Spam & content filtering
  • TLV validation
5
Translation Service

Applies client-specific or rule-based message transformations.

  • Fetches rules from Redis
  • Content modification
  • Dynamic payload mapping
6
Buffer Service

Controls timing, priority, and eligibility of message delivery.

  • Time-window enforcement
  • Country & region restrictions
  • Retry & priority handling
7
FDLR Service

Manages controlled delivery and DLR manipulation.

  • CFDLR ratio enforcement
  • Fake vs real DLR split
  • Redis reconciliation logs
8
Routing Service

Selects the optimal supplier using MCC/MNC and routing logic.

  • LCR / PCR routing
  • Choice-based percentage routing
  • Supplier rate resolution
9
SEND_SMS Service

Executes final delivery to the supplier.

  • Supplier connection handling
  • Billing trigger
  • DLR reception
10
DLR Service

Normalizes and delivers delivery reports.

  • DLR status mapping
  • SMSGID ‚Üí OMSGID mapping
  • Retry handling
11
BILLING-SMS Service

Centralized billing service used across the platform.

  • Client balance deduction
  • Supplier cost calculation
  • C-Cost & S-Cost updates
‚úÖ Final Architecture Summary

This SMPP architecture ensures secure, scalable, and traceable message delivery through containerized services, Redis-backed queues, intelligent routing, precise billing, and reliable DLR handling.
